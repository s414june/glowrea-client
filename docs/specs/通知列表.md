# Feature: 通知列表（Notifications）

## Goal

提供已登入使用者查看與自身相關的互動通知，讓使用者了解自己的帳號活動。

通知與「私人（對話）」分開：通知側重**行為事件**（喜歡、轉發、追蹤），私人側重**對話脈絡**（提及、回覆）。

---

## Scope

### In Scope

- 顯示通知列表（mention / favourite / reblog / follow）
- 首次載入資料
- 顯示 loading / empty / error state
- 支援載入更多（infinite scroll）
- 需登入才可存取，未登入導向 `/login`
- 複用 `LoadingSkeleton`、`StatusItem` 等現有元件

### Out of Scope

- 通知分類篩選（依類型）
- 已讀 / 未讀狀態標記
- 批次操作（全部標記為已讀、清除）
- Push 通知（Web Push API）
- Streaming 即時更新

---

## User Story

作為已登入使用者，我希望能看到最近有哪些人喜歡、轉發了我的貼文，或開始追蹤我，讓我能快速掌握帳號活動。

---

## User Flow

1. 使用者進入 `/notifications`
2. 頁面顯示 loading skeleton
3. API 回傳後，顯示通知列表（最新在上）
4. 列表為空時顯示 empty state
5. 向下捲動到底部時，自動載入更多舊通知
6. API 發生錯誤時，顯示 error state 與 retry 按鈕

---

## Route

### Path

- `/notifications`

### Route Guard

- 需登入；使用既有 `auth.global.ts` middleware

### Route Responsibility

- 頁面只負責初始化與渲染
- 不在 route 層處理資料轉換邏輯

## UI Requirements

### 通知項目

每則通知至少顯示：

| 欄位 | 說明 |
|------|------|
| actor 頭像 | 觸發動作的帳號 |
| actor 名稱 | 顯示名稱 + handle |
| 動作類型 | 「喜歡了你的貼文」/ 「轉發了你的貼文」/ 「開始追蹤你」/ 「提及了你」 |
| 時間 | 相對時間（例如：2 分鐘前） |
| 相關貼文預覽 | 若有（mention / favourite / reblog），顯示原貼文摘要 |

### Loading State

- 使用現有 `LoadingSkeleton` 元件

### Empty State

- 說明文字：「目前沒有通知」

### Error State

- 顯示錯誤訊息 + Retry 按鈕

### 列表

- Infinite scroll sentinel（`IntersectionObserver`），與 home timeline 一致

## 資料流

### API Endpoint

`GET /api/notifications`

query params：
- `maxId?: string`

### 對應 Mastodon API

```
GET /api/v1/notifications?limit=20[&max_id={maxId}]
```

需要 Authorization header。

### 回傳格式

```ts
type NotificationItem = {
  id: string
  type: 'mention' | 'favourite' | 'reblog' | 'follow'
  createdAt: string
  account: {
    id: string
    username: string
    acct: string
    displayName: string
    avatar: string
    url: string
  }
  status?: TimelineStatus  // mention / favourite / reblog 才有
}

type NotificationsPageResponse = {
  items: NotificationItem[]
  nextMaxId: string | null
}
```

### 分頁

Link header `rel="next"` → 解析 `max_id` 參數，與 home timeline 一致。

## Server Route 規格

### `GET /api/notifications`

- 呼叫 `requireRequestCredentials` 取得 token
- 呼叫 `GET /api/v1/notifications?limit=20[&max_id=...]`
- 帶 Authorization header
- 回傳 `NotificationsPageResponse`
- 解析 Link header 取得 `nextMaxId`

---

## 錯誤處理

| 情境 | 行為 |
|------|------|
| Mastodon API 回傳非 2xx | 回傳對應 HTTP 狀態碼給前端 |
| 未登入 | 401，middleware 導向 `/login` |
| 網路錯誤 | 前端顯示 error state + retry |
| 列表為空 | 顯示 empty state，不視為錯誤 |
