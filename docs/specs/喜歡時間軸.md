# Feature: 喜歡時間軸（Favourites）

## Goal

讓已登入使用者能查看自己喜歡（Like）過的貼文列表，作為回顧曾表達喜好之內容的入口。

喜歡與書籤的定位不同：喜歡側重情緒標記，書籤側重「稍後再看」的收藏。

---

## Scope

### In Scope

- 顯示喜歡貼文列表（`GET /api/v1/favourites`）
- 首次載入資料
- 顯示 loading / empty / error state
- 支援載入更多（infinite scroll）
- 需登入才可存取，未登入導向 `/login`
- 複用 `TimelineList`、`StatusItem`、`LoadingSkeleton` 等現有元件

### Out of Scope

- 從列表內取消喜歡的互動操作
- 喜歡分類或排序調整
- 喜歡數量統計

---

## User Story

作為已登入使用者，我希望能查看我喜歡過的貼文，以便重新找到曾經標記的內容。

---

## User Flow

1. 使用者進入 `/likes`
2. 頁面顯示 loading skeleton
3. API 回傳後，顯示喜歡貼文列表（依時間由新到舊）
4. 列表為空時顯示 empty state
5. 向下捲動到底部時，自動載入更多
6. API 發生錯誤時，顯示 error state 與 retry 按鈕

---

## Route

### Path

- `/likes`

### Route Guard

- 需登入；使用既有 `auth.global.ts` middleware

### Route Responsibility

- 頁面只負責初始化與渲染
- 不在 route 層處理資料轉換邏輯

---

## UI Requirements

### 頁面標題區

- 顯示「喜歡」標題（小型 section header）
- 不需返回按鈕

### Loading State

- 使用現有 `LoadingSkeleton` 元件

### Empty State

- 說明文字：「還沒有喜歡任何貼文」

### Error State

- 顯示錯誤訊息 + Retry 按鈕

### 列表

- 複用 `TimelineList` 元件
- Infinite scroll sentinel（`IntersectionObserver`），與 home timeline 一致

---

## 資料流

### API Endpoint

`GET /api/favourites`

query params：
- `maxId?: string`

### 對應 Mastodon API

```
GET /api/v1/favourites?limit=20[&max_id={maxId}]
```

需要 Authorization header。

### 回傳格式

複用現有型別：

```ts
TimelinePageResponse = {
  items: TimelineStatus[]
  nextMaxId: string | null
}
```

### 分頁

Link header `rel="next"` → 解析 `max_id` 參數，與 home timeline 一致。

## Server Route 規格

### `GET /api/favourites`

- 呼叫 `requireRequestCredentials` 取得 token
- 呼叫 `GET /api/v1/favourites?limit=20[&max_id=...]`
- 帶 Authorization header
- 回傳 `TimelinePageResponse`
- 解析 Link header 取得 `nextMaxId`

---

## 錯誤處理

| 情境 | 行為 |
|------|------|
| Mastodon API 回傳非 2xx | 回傳對應 HTTP 狀態碼給前端 |
| 未登入 | 401，middleware 導向 `/login` |
| 網路錯誤 | 前端顯示 error state + retry |
| 列表為空 | 顯示 empty state，不視為錯誤 |
