# Feature: 私人時間軸（Conversations）

## Goal

提供已登入使用者查看 Mastodon 私訊對話列表（Conversations），讓使用者能掌握進行中的私人對話。

v0 僅實作對話列表；點入個別對話的完整聊天室介面另立 feature。

---

## Scope

### In Scope

- 顯示對話列表（`GET /api/v1/conversations`）
- 每則對話顯示：對方頭像、名稱、最後一則訊息摘要、相對時間、未讀標記
- 首次載入資料
- 顯示 loading / empty / error state
- 支援載入更多（infinite scroll）
- 需登入才可存取，未登入導向 `/login`
- 複用 `LoadingSkeleton` 等現有元件

### Out of Scope

- 個別對話的完整聊天室介面（另立 feature）
- 已讀標記操作（`POST /api/v1/conversations/:id/read`）
- 發送新私訊
- Push / Streaming 即時更新

---

## User Story

作為已登入使用者，我希望能看到目前進行中的私人對話列表，並知道哪些對話有未讀訊息。

---

## User Flow

1. 使用者進入 `/timelines/messages`
2. 頁面顯示 loading skeleton
3. API 回傳後，顯示對話列表（最新在上）
4. 列表為空時顯示 empty state
5. 向下捲動到底部時，自動載入更多
6. API 發生錯誤時，顯示 error state 與 retry 按鈕

---

## Route

### Path

- `/timelines/messages`（既有路由，巢狀於 `/timelines` parent layout）

### Route Guard

- 需登入；使用既有 `auth.global.ts` middleware

## UI Requirements

### 對話項目

| 欄位 | 說明 |
|------|------|
| 頭像 | 對方帳號頭像（多人對話取第一位） |
| 名稱 | 對方顯示名稱 + `@handle` |
| 最後訊息 | 去除 HTML 標籤後的純文字，最多 2 行 |
| 時間 | 相對時間（例如：3 分鐘前） |
| 未讀標記 | 圓點 indicator（`unread === true`） |

### Loading State

- 使用現有 `LoadingSkeleton` 元件

### Empty State

- 說明文字：「目前沒有私人對話」

### Error State

- 顯示錯誤訊息 + Retry 按鈕

---

## 資料流

### API Endpoint

`GET /api/conversations`

query params：
- `maxId?: string`

### 對應 Mastodon API

```
GET /api/v1/conversations?limit=20[&max_id={maxId}]
```

需要 Authorization header。

### 回傳格式

```ts
type ConversationAccount = {
  id: string
  username: string
  acct: string
  displayName: string
  avatar: string
}

type ConversationItem = {
  id: string
  accounts: ConversationAccount[]
  lastStatus: TimelineStatus | null
  unread: boolean
}

type ConversationsPageResponse = {
  items: ConversationItem[]
  nextMaxId: string | null
}
```

### 分頁

Link header `rel="next"` → 解析 `max_id` 參數。

## 錯誤處理

| 情境 | 行為 |
|------|------|
| Mastodon API 回傳非 2xx | 回傳對應 HTTP 狀態碼給前端 |
| 未登入 | 401，middleware 導向 `/login` |
| 網路錯誤 | 前端顯示 error state + retry |
| 列表為空 | 顯示 empty state，不視為錯誤 |
