# Feature: 探索與搜尋（Explore）

## Goal

提供主動探索內容的入口，讓使用者不只依賴時間軸，也能搜尋帳號、標籤、貼文，或瀏覽熱門趨勢內容。

訪客與已登入使用者均可使用搜尋與趨勢瀏覽；部分結果（如完整帳號搜尋）需登入才能使用。

---

## Scope

### In Scope

- 搜尋框：輸入關鍵字後顯示結果（帳號 / 標籤 / 貼文）
- 搜尋結果分 tab 顯示：貼文、帳號、標籤
- 趨勢標籤列表（`GET /api/v1/trends/tags`）
- 顯示 loading / empty / error state
- 訪客可瀏覽趨勢，已登入使用者可全功能搜尋
- 複用 `StatusItem`、`LoadingSkeleton` 等現有元件

### Out of Scope

- 搜尋建議 / autocomplete
- AI 推薦或個人化排序
- 搜尋進階篩選（語言、日期範圍）
- 跨 instance 搜尋
- Hashtag timeline（點擊 hashtag 進入獨立頁面，另立 feature）

---

## User Story

作為使用者，我希望能在搜尋框輸入關鍵字後找到相關的貼文、帳號或標籤；作為訪客，我希望能瀏覽目前熱門的標籤，了解社群正在討論什麼。

---

## User Flow

### 搜尋流程

1. 使用者在搜尋框輸入關鍵字
2. 按下 Enter 或搜尋按鈕
3. 顯示搜尋結果，分 tab：貼文 / 帳號 / 標籤
4. 點擊結果項目導向對應頁面
5. 清空搜尋框後回到趨勢視圖

### 趨勢流程

1. 使用者進入 `/explore`
2. 顯示熱門標籤列表
3. 點擊標籤導向對應標籤頁面（`/tags/:tag`，`ComingSoon` placeholder）

---

## Route

### Path

- `/explore`

### Route Guard

- 無保護，訪客可直接存取

### Route Responsibility

- 頁面負責搜尋框狀態與 tab 切換
- 不在 route 層處理搜尋邏輯

---

## UI Requirements

### 頁面結構

```
┌──────────────────────────┐
│ 搜尋框                    │
├──────────────────────────┤
│ [貼文] [帳號] [標籤]      │  ← 有搜尋關鍵字時顯示
├──────────────────────────┤
│ 趨勢標籤列表              │  ← 無搜尋關鍵字時顯示
└──────────────────────────┘
```

### 搜尋框

- placeholder：「搜尋」
- 輸入後即時更新 tab 內容（debounce 300ms）或按 Enter 送出

### 搜尋結果 Tabs

| Tab | 顯示內容 |
|-----|---------|
| 貼文 | 匹配貼文，複用 `StatusItem` |
| 帳號 | 帳號卡片（頭像 + 名稱 + handle + 簡短描述） |
| 標籤 | 標籤名稱 + 使用次數 |

### 趨勢標籤

- 顯示 tag 名稱 + 近期使用次數
- 可點擊（導向 `/tags/:tag`，目前為 `ComingSoon`）

### Loading State

- 使用現有 `LoadingSkeleton` 元件

### Empty State

- 搜尋無結果：「找不到相關內容」
- 趨勢為空：「目前沒有熱門標籤」

### Error State

- 顯示錯誤訊息 + Retry 按鈕

## 資料流

### 搜尋 API Endpoint

`GET /api/search`

query params：
- `q: string`（必填）
- `type?: 'accounts' | 'statuses' | 'hashtags'`（省略時全類型）
- `limit?: number`（預設 20）

### 對應 Mastodon API

```
GET /api/v2/search?q={q}&type={type}&limit=20
```

已登入時帶 Authorization header；訪客不帶（搜尋功能受限）。

### 趨勢 API Endpoint

`GET /api/trends/tags`

### 對應 Mastodon API

```
GET /api/v1/trends/tags?limit=20
```

不需要 Authorization header（公開端點）。

### 回傳格式

```ts
type SearchResponse = {
  accounts: AccountSummary[]
  statuses: TimelineStatus[]
  hashtags: HashtagItem[]
}

type HashtagItem = {
  name: string
  url: string
  history: { day: string; uses: string; accounts: string }[]
}
```

## 錯誤處理

| 情境 | 行為 |
|------|------|
| 搜尋 API 回傳非 2xx | 前端顯示 error state |
| 趨勢 API 失敗 | 靜默略過，顯示空列表 |
| 關鍵字為空字串 | 不送出搜尋，顯示趨勢視圖 |
| 網路錯誤 | 前端顯示 error state + retry |
