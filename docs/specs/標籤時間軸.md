# Feature: 標籤時間軸（Hashtags）

## Goal

讓使用者（含訪客）能查看特定 hashtag 的公開貼文時間軸，並讓已登入使用者追蹤 / 取消追蹤標籤。

---

## Scope

### In Scope

- 查看特定 hashtag 的貼文時間軸（`GET /api/v1/timelines/tag/:hashtag`）
- 顯示標籤名稱、近期使用量統計
- 顯示已登入使用者是否已追蹤此標籤
- 已登入使用者可追蹤 / 取消追蹤標籤（`POST/DELETE /api/v1/tags/:name/follow`）
- Loading / empty / error state
- Infinite scroll
- 訪客可瀏覽（公開資料），追蹤功能需登入

### Out of Scope

- 全域標籤瀏覽（由探索頁處理）
- 標籤黑名單 / 過濾
- 多標籤交叉搜尋

---

## User Story

作為使用者，我希望能點擊貼文中的 hashtag 進入標籤時間軸，查看該話題下的所有公開貼文。

作為已登入使用者，我希望能追蹤標籤，讓相關貼文出現在我的首頁時間軸。

---

## User Flow

1. 使用者點擊貼文中的 `#hashtag` 連結，或從探索頁點入 → 進入 `/tags/:name`
2. 頁面顯示標籤名稱（`#name`）與近期使用量
3. 已登入使用者看到「追蹤」/「已追蹤」按鈕
4. 下方顯示貼文列表（loading → 貼文 / empty / error）
5. 向下捲動自動載入更多
6. 點擊「追蹤」→ 呼叫 follow API，按鈕切換為「已追蹤」
7. 點擊「已追蹤」→ 呼叫 unfollow API，按鈕切換回「追蹤」

---

## Route

### Path

- `/tags/:name`

### Route Guard

- 公開，訪客可瀏覽
- 追蹤 / 取消追蹤動作：需登入（前端判斷，未登入不顯示按鈕）

---

## UI Requirements

```
┌──────────────────────────┐
│ #hashtag                  │  ← 標籤名稱（大標）
│ 近 2 天 1,234 則貼文      │  ← 使用量統計
│                [追蹤]     │  ← 已登入才顯示
├──────────────────────────┤
│ 貼文列表...               │
└──────────────────────────┘
```

- 標籤名稱：headline font，顯示 `#name`
- 使用量：來自 `history[0].uses + history[1].uses`
- 追蹤按鈕：
  - 未追蹤：outline style，文字「追蹤」
  - 已追蹤：filled accent style，文字「已追蹤」
  - 追蹤中：disabled + loading 狀態
- 貼文列表：複用 `StatusItem`、`LoadingSkeleton`

## 資料流

### API Endpoints

#### 取得標籤資訊（含是否已追蹤）

`GET /api/tags/:name`

對應 Mastodon API：
```
GET /api/v1/tags/:name
```

訪客不帶 Authorization 也可呼叫（回傳 `following: false`）。

回傳格式：
```ts
type TagInfo = {
  name: string
  following: boolean
  history: { day: string; uses: string; accounts: string }[]
}
```

#### 取得標籤時間軸

`GET /api/tags/:name/timeline`

query params：
- `maxId?: string`

對應 Mastodon API：
```
GET /api/v1/timelines/tag/:name?limit=20[&max_id={maxId}]
```

訪客可呼叫（不帶 Authorization）。

回傳格式：複用 `TimelinePageResponse`

#### 追蹤標籤

`POST /api/tags/:name/follow`

對應 Mastodon API：`POST /api/v1/tags/:name/follow`

需登入。

#### 取消追蹤標籤

`DELETE /api/tags/:name/follow`

對應 Mastodon API：`DELETE /api/v1/tags/:name/follow`

需登入。

## Server Route 規格

### `GET /api/tags/:name`

- 嘗試 `requireRequestCredentials`；失敗則以匿名方式呼叫
- 呼叫 `GET /api/v1/tags/:name`
- 回傳 `TagInfo`

### `GET /api/tags/:name/timeline`

- 嘗試 `requireRequestCredentials`；失敗則以匿名方式呼叫（使用 `mastodonApiBase`）
- 呼叫 `GET /api/v1/timelines/tag/:name?limit=20[&max_id=...]`
- 解析 Link header 取得 `nextMaxId`
- 回傳 `TimelinePageResponse`

### `POST /api/tags/:name/follow` / `DELETE /api/tags/:name/follow`

- 呼叫 `requireRequestCredentials`（必須登入）
- 呼叫對應 Mastodon API
- 回傳更新後的 `TagInfo`

---

## 錯誤處理

| 情境 | 行為 |
|------|------|
| 標籤不存在 | 顯示「找不到此標籤」error state |
| 追蹤 / 取消追蹤失敗 | 按鈕恢復原狀態，顯示錯誤提示 |
| 時間軸載入失敗 | 顯示 error state + retry |
| 訪客點追蹤 | 不顯示按鈕（前端控制），不會觸發此情境 |
