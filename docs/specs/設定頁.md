# Feature: 設定頁（Settings）

## Goal

提供基本設定入口，讓使用者可調整外觀與偏好，不增加複雜操作。v0 以本地儲存優先，不需後端 API。

---

## Scope

### In Scope

- 設定頁面骨架（tab 結構：外觀 / 語言 / 關於）
- 外觀設定：色彩主題（淺色 / 深色 / 跟隨系統）
- 語言設定：介面語言（v0 固定提供繁體中文；預留其他語言位置）
- 關於：應用程式版本、連結（原始碼、實例說明）
- 設定值儲存於 `localStorage`（client-side only）
- 需登入才可存取，未登入導向 `/login`

### Out of Scope

- 帳號設定（修改個人資料、密碼）
- 通知偏好設定
- 隱私設定
- 多語言 i18n 完整實作
- 設定同步至 Mastodon 後端

---

## User Story

作為已登入使用者，我希望能切換應用程式的深淺色主題，讓介面符合我的使用習慣。

---

## User Flow

1. 使用者進入 `/settings`
2. 顯示設定頁，預設停在「外觀」tab
3. 使用者切換色彩主題 → 立即套用，儲存至 localStorage
4. 使用者切換 tab → 顯示對應設定區塊
5. 離開頁面後重新進入，設定值從 localStorage 還原

---

## Route

### Path

- `/settings`

### Route Guard

- 需登入；使用既有 `auth.global.ts` middleware

### Route Responsibility

- 頁面負責 tab 切換與設定值讀寫
- 不呼叫後端 API（v0）

## 資料流

### 設定儲存

所有設定值儲存於 `localStorage`，key 前綴統一為 `glowrea:`。

```ts
type AppSettings = {
  theme: 'light' | 'dark' | 'system'
  locale: 'zh-TW'  // v0 固定
}
```

預設值：

```ts
const defaultSettings: AppSettings = {
  theme: 'system',
  locale: 'zh-TW',
}
```

### 主題套用

- `system`：監聽 `prefers-color-scheme` media query，自動切換
- `light` / `dark`：在 `<html>` 加上 class（`class="light"` 或 `class="dark"`）
- SSR 期間以 `system` 為預設，避免閃爍（hydration mismatch）

## Composable 規格：`useAppSettings`

```ts
type AppSettings = {
  theme: 'light' | 'dark' | 'system'
  locale: 'zh-TW'
}

type AppSettingsState = {
  settings: Ref<AppSettings>
  setTheme: (theme: AppSettings['theme']) => void
}
```

- `setTheme` 寫入 localStorage 並立即更新 `<html>` class
- 僅在 `import.meta.client` 下存取 localStorage

---

## 錯誤處理

| 情境 | 行為 |
|------|------|
| localStorage 不可用 | 靜默略過，使用預設值，不顯示錯誤 |
| localStorage 值格式錯誤 | 重設為預設值 |
